Step-by-Step Guide: Creating Notifications for Document Updates or New Uploads

For Users:

1. Access Power Automate

  • Log in to Power Automate using your Microsoft account
  • Ensure you have the necessary permissions for your SharePoint site

2. Create a New Flow

  • Click on "Create" in the left sidebar
  • Select "Automated cloud flow"

3. Choose a Trigger

  • Search for "SharePoint" in the trigger search box
  • Select "When a file is created or modified (properties only)" for SharePoint

4. Configure the Trigger

  • Select the Site Address and Library Name where you want to monitor changes
  • Choose whether to trigger on new items, modified items, or both

5. Add an Action

  • Click "New step" and search for your desired action:
  • For Teams: Choose "Post a message in a chat or channel"
  • For Outlook: Choose "Send an email (V2)"

6. Configure the Action

  • For Teams: Select the team and channel, compose your message
  • For Outlook: Enter recipient emails, compose subject and body

7. Add Conditions (Optional)

  • Click "New step" and add a condition to filter notifications based on specific criteria

8. Save and Test the Flow

  • Click "Save" to save your flow
  • Use the "Test" button to run a test and ensure the flow works as expected

9. Monitor and Refine

  • Regularly check the flow's run history to ensure it's working correctly
  • Adjust as needed based on user feedback and changing requirements

For AI System:

1. Create a SharePoint List for AI Notifications

  • In your SharePoint site, create a new list to store notifications for the AI system

2. Modify the Flow

  • Follow steps 1-4 from the user notification process
  • Add a new action: "Create item" (SharePoint)
  • Configure it to create an item in your AI Notifications list with relevant details

3. Set Up AI Integration

  • Work with your AI development team to create an API or webhook that can read from the SharePoint list
  • Ensure the AI system can process and act on the notifications as needed

4. Test and Monitor

  • Test the flow to ensure items are being created in the SharePoint list
  • Verify that the AI system is correctly receiving and processing the notifications

Best Practices

  • Use clear and descriptive names for your flows
  • Document your flows and share the documentation with relevant team members
  • Regularly review and optimize your flows to ensure they meet evolving business needs
  • Set up error notifications to alert you if a flow fails
  • Use Flow checker to identify and resolve potential issues in your flows
